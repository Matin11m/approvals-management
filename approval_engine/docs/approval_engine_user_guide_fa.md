# راهنمای استفاده از Approval Engine (Odoo 19)

## 1) این اپ کجاست؟
بعد از نصب ماژول:
1. وارد App Launcher شوید.
2. اپ **Approval Engine** را باز کنید.
3. منوهای اصلی را می‌بینید:
   - `Requests`
   - `Approvals Forms`
   - `Templates`

---

## 2) سناریوی استاندارد راه‌اندازی

### قدم 1: بررسی فرم‌های Approvals
- مسیر: `Approval Engine > Approvals Forms`
- فرم‌هایی که قبلاً در ماژول approvals ساخته‌اید اینجا قابل مشاهده‌اند.

### قدم 2: ساخت/تکمیل Template
- مسیر: `Approval Engine > Templates`
- یک template را باز/ایجاد کنید.
- فیلدها را کامل کنید:
  - `Approvals Form`: فرم مبدا
  - `Model`: مدل مقصد (مثلاً Purchase Order)
  - `Active`: بعد از تکمیل روشن شود

### قدم 3: تعریف مراحل تایید
در تب **Steps**:
- برای هر مرحله:
  - نام مرحله
  - گروه تاییدکننده
  - حداقل تعداد تایید
  - در صورت نیاز، الزام تایید همه اعضای گروه

### قدم 4: اتصال به View و Function
در تب **UI Bindings**:
- `view_id`: ویوی هدف از همان مدل
- `trigger_method`: تابعی که باید برای ارسال تایید صدا زده شود
- `button_label`: متن دکمه

> نکته: Binding در وضعیت فعلی ساختار نگاشت را نگه می‌دارد؛ تزریق خودکار دکمه در view نیازمند فاز بعدی توسعه است.

### قدم 5: تعریف Rule (اختیاری)
- اگر همه رکوردها باید تایید شوند: `Always`
- اگر شرط دامنه‌ای می‌خواهید: `Domain`
- اگر منطق پیچیده می‌خواهید: `Python Expression`

### قدم 6: تست عملی
- روی رکورد مدل مقصد، متد ارسال برای تایید را صدا بزنید.
- در `Requests` وضعیت را پیگیری کنید.
- تصمیم تایید/رد را ثبت کنید و لاگ‌ها را بررسی کنید.

---

## 3) نمونه عملی سریع (Purchase)
1. یک template با `Model = purchase.order` بسازید.
2. در Steps، گروه مدیر خرید را مرحله اول بگذارید.
3. در Binding، view فرم PO را انتخاب کنید.
4. trigger_method را `action_submit_for_approval` بگذارید.
5. template را Active کنید.

---

## 4) خطاهای رایج
- **Template active است ولی Step ندارد**
  - حداقل یک step اضافه کنید.
- **Rule type روی Domain/Python است ولی مقدار خالی است**
  - فیلد مربوط را تکمیل کنید.
- **نام method نامعتبر است**
  - فقط identifier معتبر پایتون وارد کنید.

---

## 5) بهترین شیوه استفاده
- ابتدا templateها را inactive نگه دارید و کامل تست کنید.
- برای هر مدل، نام‌گذاری template شفاف داشته باشید.
- از groupهای واقعی فرآیند سازمان استفاده کنید.
- قبل از فعال‌سازی در production، یک Pilot روی یک مدل اجرا کنید.


## 6) نکته مهم درباره دسترسی‌ها
در نسخه فعلی، برای جلوگیری از پیچیدگی زودهنگام، یک دسترسی پایه برای کاربران داخلی فعال شده تا منوها/عملیات اصلی قابل استفاده باشد.
قبل از استفاده production، حتماً تفکیک نقش، ACL دقیق و Record Ruleهای نهایی اضافه شود.
